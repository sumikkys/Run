# Run! 前端概念页

这是基于项目 README 和完整 Proposal 文档制作的移动端概念原型，聚焦主 Demo：

```text
已购订单正常可赶上
→ 第一次提醒
→ 忽略后原站方案失效
→ 搜索下游站拦截与补救方案
→ 输出最小损失行动卡
```

## 打开方式

直接用浏览器打开：

```text
src/frontend/index.html
```

不需要安装依赖，也不需要启动开发服务器。

## 页面内容

- `订单`：展示已购车票、当前位置、ETA 和进站缓冲。
- `提醒`：模拟第一次 Run! 提醒，用户仍可赶上原车。
- `救急`：模拟忽略 35 分钟后原站方案失效，开始搜索补救方案。
- `行动卡`：展示“下游站拦截原车”的最小损失方案。
- `异常控制台`：用于路演时模拟 ETA 增加、排队增加、余票消失、原车晚点。
- `实验室`：自定义车次救急 Beta 的概念入口。
